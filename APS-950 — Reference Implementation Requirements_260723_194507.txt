───

1. Purpose
APS-950 definiuje wymagania dla implementacji referencyjnych Aura Protocol.
Implementacja referencyjna stanowi wzorzec zgodności dla wszystkich przyszłych implementacji.

───

2. Objectives
Reference Implementation MUST:
• implementować wszystkie obowiązkowe wymagania APS,
• przechodzić pełny zestaw testów Conformance,
• generować poprawny Evidence Pack,
• zachowywać pełną zgodność z Protocol Invariants,
• umożliwiać niezależny audyt.

5. Coding Requirements
Implementacja MUST:
• być deterministyczna,
• być możliwa do odtworzenia,
• nie naruszać Protocol Invariants,
• posiadać pełne testy jednostkowe,
• posiadać testy zgodności APS-400.

───

6. Build Requirements
Proces budowania MUST:
• być powtarzalny,
• być wersjonowany,
• generować identyczne artefakty dla tej samej wersji źródła,
• dokumentować użyte zależności.

───

7. Test Requirements
Przed wydaniem implementacja MUST przejść:
• Unit Tests,
• Integration Tests,
• Conformance Tests,
• Replay Tests,
• Evidence Validation,
• Security Validation.

───

8. Documentation Requirements
Każda implementacja MUST zawierać:
• README,
• instrukcję uruchomienia,
• opis architektury,
• mapowanie do APS,
• listę wspieranych wersji protokołu.

───

9. Release Requirements
Każde wydanie MUST zawierać:
• numer wersji,
• Compatibility Statement,
• Conformance Report,
• Evidence Pack,
• Release Notes.

───

10. Certification
Implementacja może zostać oznaczona jako Aura Reference Implementation, jeżeli:
• spełnia wszystkie wymagania APS-950,
• uzyskała wynik PASS dla wszystkich obowiązkowych testów APS-400,
• posiada kompletną dokumentację,
• przeszła Architecture Review.

───

12. Governance
Zmiany w implementacjach referencyjnych:
• MUST być zgodne z APS,
• MUST przejść Architecture Review,
• MUST zachować zgodność z obowiązującymi Protocol Invariants,
• MUST być udokumentowane w ADR oraz CHANGELOG.
