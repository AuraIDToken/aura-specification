# Code of Conduct

## Our Pledge

Contributors to `aura-specification` pledge to make participation in this project a harassment-free experience for everyone, regardless of background, experience level, nationality, personal appearance, religion, or other characteristic.

## Our Standards

Acceptable behaviour:
- Demonstrating technical precision and intellectual honesty
- Respectfully challenging specifications with evidence and reasoned argument
- Accepting that the Chief Architect holds final authority over canonical documents
- Crediting prior work and acknowledging sources
- Raising concerns via the formal RFC or ADR process rather than unilateral changes

Unacceptable behaviour:
- Modifying frozen or approved documents without the Amendment Procedure
- Bypassing the RFC / ADR review process
- Introducing undocumented or speculative protocol behaviour
- Personal attacks, harassment, or discriminatory language

## Enforcement

Violations may result in removal of contributions or loss of contributor access. Final decisions rest with the Chief Architect as defined in the AURA Constitution Article VIII.

## Scope

This Code of Conduct applies to all spaces managed by this project: GitHub issues, pull requests, discussions, and any other project forum.
